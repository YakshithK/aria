## Failure Modes Per Codepath

### Window Registry
- `EnumWindows` returns partial list if a window opens during enumeration → registry is a snapshot, accept this
- UIA window event subscription fails silently on some Windows builds → fall back to periodic poll (5s)
- process_name lookup fails for protected processes → classify as `unsupported`, log warning

### CDP Backend
- `/json/list` returns empty (app not launched with debug port) → raise `CDPNotAvailableError` with instructions
- WebSocket disconnects mid-observation → reconnect with exponential backoff (max 3 retries)
- `getFullAXTree` times out (>5s) → raise `ObservationTimeoutError`, conductor returns last cached map
- Port collision (two apps same port) → raise `PortConflictError` at registration time, not at observation time
- Discord virtual list: messages not in DOM → scroll-to-bottom + re-observe loop required (see BUILD_ORDER step 5)

### Planner Loop
- `finish_reason == "tool_calls"` with no matching tool definition → log error, return structured failure
- `tool_call_id` mismatch in history → Ollama returns 400; history format must be validated before each call
- max_turns exceeded → return `{"status": "max_turns"}`, do not raise exception (allow caller to retry)
- Ollama not running → catch `ConnectionRefusedError`, surface "run `ollama serve` first"
- Context window exceeded (large SemanticMap) → reduce element cap and retry observation

### Action Executor
- `Runtime.callFunctionOn` on a detached frame → CDP returns error; executor retries with re-observed target
- `force_foreground` fails (SetForegroundWindow returns False) → retry once after 100ms; log if still failing
- COM apartment violation (wrong thread) → always use the dedicated COM ThreadPoolExecutor(max_workers=1)

## Parallelization Strategy (Worktrees)

Tasks within a BLOCK that don't share files can run in parallel worktrees:

```
BLOCK 0: T01-T02 parallel, then T03, then T04-T05 parallel
BLOCK 1: T07-T09 parallel, then T10-T11 sequential
BLOCK 2: T12-T15 sequential (CDP builds on connect), T16 parallel with T17
BLOCK 3: T18-T22 sequential (planner builds incrementally), T23 parallel with T24
BLOCK 4-7: sequential per block (each depends on prior)
```

Cross-block gate: CI must be green before starting the next block.
